**Content to Add:**
```
Title: EMI Calculator - Financial Planning Tool

PURPOSE:
━━━━━━━
Help customers understand affordability
Break down total investment into monthly payments
Reduce purchase hesitation through transparency

HOW EMI CALCULATION WORKS:

Mathematical Formula:
═══════════════════════════════════════════
EMI = (P × r × (1+r)^n) / ((1+r)^n - 1)

Where:
  P = Principal Amount (Bike Price)
  r = Monthly Interest Rate (Annual Rate / 12 / 100)
  n = Total Number of Months (Years × 12)

EXAMPLE CALCULATION:
═══════════════════════════════════════════
Bike Price:     ₹21.50 Lakh
Tenure:         5 Years (60 Months)
Interest Rate:  8% per annum (0.67% monthly)

Calculate EMI ➜ ₹44,850 per month (approximately)

OUTPUT BREAKDOWN:
├─ Monthly EMI:        ₹44,850
├─ Total Amount Due:   ₹26,91,000
└─ Total Interest Paid: ₹5,41,000

USER INPUTS:
✓ Bike Price Field
✓ Tenure in Years (1-10 year options)
✓ Interest Rate Field
✓ Calculate Button

FEATURES:
✓ Real-time calculation
✓ Input validation
✓ Error handling for invalid inputs
✓ Professional result card
✓ Formatted currency display
✓ Clear breakdown of costs

**Content to Add:**
```
Title: Challenges Encountered & Solutions

CHALLENGE 1: RESPONSIVE VIDEO BACKGROUNDS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Problem:
  • Videos don't scale responsively natively
  • File sizes can be large (slow loading)
  • Fallback needed for unsupported browsers

Solution:
  • Used CSS object-fit property
  • Optimized video compression
  • Provided fallback image
  • Preload and autoplay attributes
  • muted attribute for autoplay compliance

CHALLENGE 2: CALCULATION ACCURACY IN EMI CALCULATOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Problem:
  • Floating-point precision errors in JavaScript
  • Complex financial formula implementation
  • Need for accurate monetary calculations

Solution:
  • Used proper formula implementation
  • Proper rounding to 2 decimals (toFixed)
  • Thorough testing with multiple values
  • Input validation before calculation
  • Clear display of all components

CHALLENGE 3: MOBILE LAYOUT COMPLEXITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Problem:
  • Complex desktop layouts don't fit on mobile

  • Too much information on small screens

Solution:
  • Implemented mobile-first CSS
  • Bootstrap responsive grid system
  • Media queries for layout changes
  • Larger touch targets on mobile
  • Progressive disclosure of information

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
